# 从微信云开发迁移到阿里云数据库操作指南

当前小程序使用**微信云开发**：云函数（login、getReservedSeats、reserveSeat、releaseSeat、renewSeat、startTempLeave、endTempLeave）+ 微信云数据库（集合 seat_locks、user_active、user_state、reservation_history）。  
若要将数据与逻辑迁到**阿里云**，需在阿里云上自建**后端 API + 数据库**，小程序端改为通过 **HTTPS 请求**调用你的接口。

---

## 一、整体架构对比

| 当前（微信云开发） | 迁移后（阿里云） |
|-------------------|------------------|
| 微信云函数 | 你自己在阿里云上的 **HTTP API 服务**（如 Node.js / Java 等） |
| 微信云数据库 | **阿里云数据库**（如 RDS MySQL、PolarDB、表格存储等） |
| 小程序里 `wx.cloud.callFunction()` | 小程序里 `wx.request()` 请求你的 API 域名 |
| 云函数内自动拿到 openid | 后端用 **code2session** 用 code 换 openid，再做鉴权 |

小程序端仍用 **wx.login()** 拿到 `code`，把 `code` 发给你的阿里云后端；后端用微信接口 `code2session`（需 AppID + AppSecret）换 `openid`，作为用户标识，和现在逻辑一致。

---

## 二、阿里云侧需要准备什么

### 1. 数据库（任选其一即可）

- **云数据库 RDS（MySQL / PostgreSQL）**：适合表结构固定、需要事务（如抢座、续约、暂离）。  
- **表格存储 TableStore**：适合简单键值/宽表，接口与微信云数据库不同，需在后端做一层封装。  

推荐先用 **RDS MySQL**，表结构容易和当前云数据库集合对应。

### 2. 后端服务（任选其一即可）

- **云服务器 ECS**：自己用 Node.js / Java / Python 等写 HTTP 接口，连接 RDS。  
- **函数计算 FC**：按请求执行，写多个“接口函数”，每个对应一个云函数逻辑；数据库连 RDS 或 TableStore。  
- **Serverless 应用引擎 SAE**：部署一个 Web 应用，暴露 REST API，连接 RDS。  

推荐：**ECS + Node.js（或 Java）** 或 **函数计算 FC**，二者都能对接微信 code2session 和你的业务表。

### 3. 网络与域名

- 为你的后端提供 **HTTPS 访问**（微信小程序要求 request 必须走 HTTPS）。  
- 使用**已备案的域名**（如 `https://api.你的域名.com`），并在小程序后台配置为 **request 合法域名**。

## 三、数据库表设计建议（MySQL 示例）

可与当前云开发集合对应建表，例如：

```sql
-- 座位占用锁（对应 seat_locks）
CREATE TABLE seat_locks (
  seat_number INT PRIMARY KEY,
  user_id VARCHAR(64) NOT NULL,
  start_time BIGINT,
  expire_time BIGINT NOT NULL,
  hours INT,
  updated_at BIGINT,
  INDEX idx_expire (expire_time)
);

-- 用户当前预约（对应 user_active）
CREATE TABLE user_active (
  user_id VARCHAR(64) PRIMARY KEY,
  seat_number INT NOT NULL,
  expire_time BIGINT,
  updated_at BIGINT
);

-- 用户状态：24h 预约次数、违规等（对应 user_state）
CREATE TABLE user_state (
  user_id VARCHAR(64) PRIMARY KEY,
  reservation_starts TEXT COMMENT 'JSON 数组，24h内预约开始时间',
  violation_until BIGINT DEFAULT 0,
  updated_at BIGINT
);

-- 预约历史（对应 reservation_history，可选）
CREATE TABLE reservation_history (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  user_id VARCHAR(64),
  seat_number INT,
  start_time BIGINT,
  expire_time BIGINT,
  hours INT,
  action VARCHAR(32),
  created_at BIGINT
);
```

业务逻辑（抢座、退坐、续约、暂离）在**后端接口**里用 SQL/事务实现，和当前云函数里对集合的增删改查一一对应即可。

---

## 四、后端接口设计建议（与现有云函数对应）

后端需提供至少以下接口（请求/响应格式可自定，这里仅作示例）：

| 云函数 | 建议的 HTTP 接口 | 说明 |
|--------|------------------|------|
| login | `POST /api/login`，body: `{ code }`，返回 `{ openid }` | 用 code 调微信 code2session，返回 openid |
| getReservedSeats | `GET /api/seats/reserved?startSeat=1&endSeat=600&now=xxx` | 返回未过期占用列表，格式与现 result.list 一致 |
| reserveSeat | `POST /api/seats/reserve`，body: `{ seatNumber, hours }`，header 带 openid 或 token | 事务：占座 + 写 user_active + user_state |
| releaseSeat | `POST /api/seats/release`，body: `{ seatNumber }`，header 带 openid | 释放座位、清 user_active |
| renewSeat | `POST /api/seats/renew`，body: `{ seatNumber, hours, expiredHours }`，header 带 openid | 续约：延长 expire_time 等 |
| startTempLeave | `POST /api/seats/tempLeave/start`，body: `{ seatNumber }`，header 带 openid | 标记暂离开始、记录时间 |
| endTempLeave | `POST /api/seats/tempLeave/end`，body: `{ seatNumber }`，header 带 openid | 取消暂离 |

- **鉴权**：小程序登录后把 `openid` 存起来，请求时放在 **header**（如 `X-Openid`）或先调 `/api/login` 换 **token**，后续接口带 token，后端再解析出 openid。  
- **code2session**：仅在后端用 AppID + AppSecret 调用微信 `https://api.weixin.qq.com/sns/jscode2session`，不要把 AppSecret 写进小程序。

## 五、小程序端需要改的地方

### 1. 配置后端地址

在项目里增加**接口根地址**配置（例如 `config.js` 或 `app.js` 的 globalData），例如：

```js
// 使用阿里云时填写你的 API 根地址（HTTPS、已备案域名）
const API_BASE = 'https://api.你的域名.com';
```

并在微信小程序后台 **开发 → 开发管理 → 开发设置 → 服务器域名** 中，把该域名加入 **request 合法域名**。

### 2. 登录改为请求你的后端

- 去掉或绕过 `wx.cloud.callFunction({ name: 'login' })`。  
- 先 `wx.login()` 取 `code`，再 `wx.request()` 调你的 `POST /api/login`，传 `{ code }`，后端返回 `openid`。  
- 小程序把返回的 `openid` 存到 `globalData.currentUserId` 和本地缓存（与现有一致），后续逻辑可不变。

### 3. 所有云函数改为 wx.request

把原来的：

- `wx.cloud.callFunction({ name: 'getReservedSeats', data: { startSeat, endSeat, now } })`  
  改为：`GET /api/seats/reserved?startSeat=...&endSeat=...&now=...`，header 带 openid（或 token）。
- `wx.cloud.callFunction({ name: 'reserveSeat', data: { seatNumber, hours } })`  
  改为：`POST /api/seats/reserve`，body `{ seatNumber, hours }`，header 带 openid。
- 同理：releaseSeat、renewSeat、startTempLeave、endTempLeave 都改成对应对的 **POST/GET + 路径 + 参数**。

请求时统一使用 **HTTPS + 上面配置的 API_BASE**，这样所有数据就都走阿里云后端和阿里云数据库了。

### 4. 错误与提示

- 网络错误、超时、4xx/5xx 可在 `wx.request` 的 fail / success 里统一处理，和现在云函数 fail 时提示“网络繁忙”等一致即可。  
- 业务错误（如“该座位已被占用”）由后端返回统一结构（如 `{ ok: false, msg: '...' }`），前端根据 `ok` 或 `msg` 弹窗提示。

## 六、操作步骤小结（按顺序做）

1. **在阿里云购买/开通**：RDS（MySQL） + 一台 ECS 或 函数计算 FC。  
2. **建表**：在 RDS 里按上面建好 `seat_locks`、`user_active`、`user_state`、`reservation_history` 等表。  
3. **写后端服务**：在 ECS 或 FC 上实现 `/api/login` 以及 getReservedSeats、reserveSeat、releaseSeat、renewSeat、startTempLeave、endTempLeave 对应的接口，逻辑与现有云函数一致，只是改为访问 MySQL。  
4. **配置 HTTPS 与域名**：为后端绑定域名并配置 SSL，确保 `https://api.你的域名.com` 可访问。  
5. **小程序配置**：在 `config` 或 `globalData` 里填 `API_BASE = 'https://api.你的域名.com'`；在小程序后台把该域名设为 request 合法域名。  
6. **小程序改代码**：  
   - 登录：wx.login + 请求 `/api/login`，保存 openid。  
   - 其余全部从 `wx.cloud.callFunction` 改为 `wx.request` 调用对应接口，并带上 openid（或 token）。  
7. **测试**：从登录到选座、抢座、退坐、续约、暂离全流程走一遍，确认数据都落在阿里云 RDS。

按以上步骤即可把数据库从微信云开发迁到阿里云；若你提供当前云函数的具体入参和返回格式，我可以按你的后端语言（如 Node 或 Java）写出对应的接口示例和表操作 SQL。
